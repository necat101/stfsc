# STFSC FuckScript Ruleset

This file is for AI agents and humans who write STFSC scripts.

Write scripts like a tired small local model must understand them. Be plain.
Do not be clever. Do not invent helpers. Use short names. Use one behavior per
script when possible.

## 0. The Tiny Mental Model

There are two kinds of FuckScript work:

1. Native Rust scripts.
   These run now. They implement the Rust trait `FuckScript`.

2. Editable FuckScript source.
   This is saved in `ScriptComponent.source` and exported into the native script
   cache. Live preview still runs the registered runtime script name, usually
   `CustomFuckScript`, until generated native bindings are connected.

Important: do not claim editable source is interpreted live. It is preserved and
exported. Native Rust presets are what actually run today.

## 1. Hard Rules

- Max scripts on one entity: `MAX_SCRIPTS_PER_ENTITY == 32`.
- Script order matters. Keep the user's order.
- Do not remove existing scripts unless the user asks.
- Prefer `SetScripts { id, names }` for editor deploys.
- Use enabled script names only in `SetScripts`.
- Unknown script names are ignored by the runtime registry.
- Keep `script_components`, `script`, and `scripts` in sync for scene files.
- New native presets must be registered and added to the built-in name list.
- Use `ScriptContext`. Do not reach around it unless native Rust code must.

## 3. Runtime Hook Order

Each enabled script slot runs in stack order.

For each script, the engine does this:

```text
if script has not started:
    on_start(ctx)
    on_enable(ctx)

on_xr_input(ctx, event) for each XR event
on_xr_frame(ctx, xr_snapshot)
on_animation_event(ctx, event_name) for each animation event
on_fixed_update(ctx)
on_update(ctx)
on_late_update(ctx)
```

Simple rule:

- Use `on_start` for setup.
- Use `on_update` for normal gameplay.
- Use `on_fixed_update` for physics.
- Use `on_late_update` for head/hand/camera following.
- Use collision hooks for collision behavior.
- Use XR hooks for controller edges and XR pose data.

## 4. Allowed Hook Names

Use only these hook names:

```text
on_start(ctx)
on_update(ctx)
on_fixed_update(ctx)
on_late_update(ctx)
on_enable(ctx)
on_disable(ctx)
on_collision_start(ctx, other)
on_collision_end(ctx, other)
on_trigger_start(ctx, other)
on_trigger_end(ctx, other)
on_player_respawn(ctx)
on_ui_event(ctx, event)
on_animation_event(ctx, event_name)
on_xr_frame(ctx, xr)
on_xr_input(ctx, event)
```

Do not write `Update`, `tick`, `awake`, `ready`, `process`, `Start`, or Unity
style names.

## 5. Native Rust Trait

Native scripts implement this trait in `src/world/scripting.rs`:

```rust
pub trait FuckScript: Send + Sync {
    fn on_start(&mut self, _ctx: &mut ScriptContext) {}
    fn on_update(&mut self, _ctx: &mut ScriptContext) {}
    fn on_fixed_update(&mut self, _ctx: &mut ScriptContext) {}
    fn on_late_update(&mut self, _ctx: &mut ScriptContext) {}
    fn on_enable(&mut self, _ctx: &mut ScriptContext) {}
    fn on_disable(&mut self, _ctx: &mut ScriptContext) {}
    fn on_collision_start(&mut self, _ctx: &mut ScriptContext, _other: Entity) {}
    fn on_collision_end(&mut self, _ctx: &mut ScriptContext, _other: Entity) {}
    fn on_trigger_start(&mut self, _ctx: &mut ScriptContext, _other: Entity) {}
    fn on_trigger_end(&mut self, _ctx: &mut ScriptContext, _other: Entity) {}
    fn on_player_respawn(&mut self, _ctx: &mut ScriptContext) {}
    fn on_ui_event(&mut self, _ctx: &mut ScriptContext, _event: &crate::ui::UiEvent) {}
    fn on_animation_event(&mut self, _ctx: &mut ScriptContext, _event_name: &str) {}
    fn on_xr_frame(&mut self, _ctx: &mut ScriptContext, _xr: &XrInputSnapshot) {}
    fn on_xr_input(&mut self, _ctx: &mut ScriptContext, _event: &XrInputEvent) {}
}
```

Only implement the hooks you need.

## 7. Editable Source Syntax

Editable source is intentionally tiny. Use this shape:

```fuckscript
script CustomFuckScript {
    on_start(ctx) {
    }

    on_update(ctx) {
    }
}
```

Small-agent syntax rules:

- Always start with `script Name {`.
- Always close braces.
- Use `//` comments.
- Use semicolons after assignments and calls.
- Use `ctx.state.name` for script memory.
- Use `ctx.transform.position`, `ctx.transform.rotation`, and
  `ctx.transform.scale` for simple movement.
- Use `ctx.dt` for time.
- Use numbers like `1.0`, not `1`.
- Prefer one `if` at a time.
- Avoid nested logic if possible.
- Avoid one-line tricks.
- Avoid ternary expressions.
- Avoid custom syntax.

Approved value helpers for editable source:

```text
vec3(x, y, z)
quat_identity()
quat_from_rotation_y(yaw)
sin(x)
atan2(y, x)
length(v)
length_squared(v)
normalize(v)
clamp(value, min, max)
slerp(a, b, t)
```

Approved context helper names for editable source:

```text
ctx.log(text)
ctx.despawn_self()
ctx.action_pressed(name)
ctx.action_just_pressed(name)
ctx.action_just_released(name)
ctx.action_value(name)
ctx.request_haptic(hand, amplitude, seconds)
ctx.track_transform_to_xr_pose(space, position_offset, rotation_offset)
```

Use hand strings:

```text
"left"
"right"
```

Use XR pose strings:

```text
"head"
"left_grip"
"right_grip"
"left_aim"
"right_aim"
```

If a helper is not in this file, do not invent it.

## 9. Simple Editable Examples

### Move Up Slowly

```fuckscript
script MoveUp {
    on_update(ctx) {
        let p = ctx.transform.position;
        let speed = 1.0;

        ctx.transform.position = vec3(p.x, p.y + speed * ctx.dt, p.z);
    }
}
```

### Spin Around

```fuckscript
script Spin {
    on_start(ctx) {
        ctx.state.yaw = 0.0;
    }

    on_update(ctx) {
        ctx.state.yaw = ctx.state.yaw + 1.5 * ctx.dt;
        ctx.transform.rotation = quat_from_rotation_y(ctx.state.yaw);
    }
}
```

### Bounce Up And Down

```fuckscript
script Bounce {
    on_start(ctx) {
        ctx.state.time = 0.0;
        ctx.state.base_y = ctx.transform.position.y;
    }

    on_update(ctx) {
        ctx.state.time = ctx.state.time + ctx.dt;

        let p = ctx.transform.position;
        let y = ctx.state.base_y + sin(ctx.state.time * 2.0) * 0.5;

        ctx.transform.position = vec3(p.x, y, p.z);
    }
}
```

### Destroy Self On Collision

```fuckscript
script DestroyOnTouch {
    on_collision_start(ctx, other) {
        ctx.log("hit");
        ctx.despawn_self();
    }
}
```

### Pulse Right Controller

```fuckscript
script PulseRightTrigger {
    on_update(ctx) {
        if ctx.action_just_pressed("fire") {
            ctx.request_haptic("right", 0.7, 0.08);
        }
    }
}
```

### Follow Right Hand

Use `on_late_update` for hand/head following.

```fuckscript
script FollowRightHand {
    on_late_update(ctx) {
        ctx.track_transform_to_xr_pose(
            "right_grip",
            vec3(0.0, 0.0, 0.0),
            quat_identity()
        );
    }
}
```

### Open A Door Slowly

```fuckscript
script OpenDoorOnTouch {
    on_start(ctx) {
        ctx.state.open = false;
        ctx.state.angle = 0.0;
    }

    on_trigger_start(ctx, other) {
        ctx.state.open = true;
    }

    on_update(ctx) {
        if ctx.state.open == true {
            ctx.state.angle = ctx.state.angle + 1.5 * ctx.dt;
            ctx.state.angle = clamp(ctx.state.angle, 0.0, 1.5708);
            ctx.transform.rotation = quat_from_rotation_y(ctx.state.angle);
        }
    }
}
```

## 10. UI Rules

UI layout callbacks are handled by the engine. The built-in callback strings are:

```text
menu_load("alias")
menu_back()
menu_back
```

Do not write `ctx.menu_back()` in a script unless a native helper is added.

Scripts can still receive `on_ui_event(ctx, event)`, but the current native
`ScriptContext` does not expose a simple menu helper. For menus, prefer UI
layout callbacks.

Simple UI source example:

```fuckscript
script LogUiClick {
    on_ui_event(ctx, event) {
        ctx.log("ui event");
    }
}
```

## 11. Physics Rules

Use `on_fixed_update` for physics.

Simple movement of non-physics objects may change `ctx.transform`.

Dynamic physics objects should use native Rust today. Editable source may show
intent, but the real force call must map to `ctx.physics` and
`RigidBodyHandle` in Rust.

Bad for physics objects:

```fuckscript
ctx.transform.position = ctx.transform.position + vec3(0.0, 10.0, 0.0);
```

Better idea:

```text
Make a native Rust preset that gets RigidBodyHandle and calls body.add_force().
```

## 13. Native Rust Examples

### Native No-Op Script

```rust
pub struct MyScript;

impl FuckScript for MyScript {}
```

### Native Move Up

```rust
pub struct MoveUpScript;

impl FuckScript for MoveUpScript {
    fn on_update(&mut self, ctx: &mut ScriptContext) {
        let dt = ctx.dt;

        if let Some(mut transform) = ctx.transform_mut() {
            transform.position.y += 1.0 * dt;
        }
    }
}
```

### Native Face Player

This is the safe Rust pattern:

1. Read player position first.
2. Drop the query borrow.
3. Mutate this entity.

```rust
pub struct FacePlayerScript;

impl FuckScript for FacePlayerScript {
    fn on_update(&mut self, ctx: &mut ScriptContext) {
        let mut player_pos = None;

        for (_entity, (transform, _player)) in
            ctx.world.query::<(&Transform, &Player)>().iter()
        {
            player_pos = Some(transform.position);
            break;
        }

        let Some(player_pos) = player_pos else {
            return;
        };

        let dt = ctx.dt;

        if let Some(mut transform) = ctx.transform_mut() {
            let to_player = player_pos - transform.position;

            if to_player.length_squared() > 0.001 {
                let yaw = to_player.x.atan2(to_player.z);
                let target = glam::Quat::from_rotation_y(yaw);
                let amount = (dt * 8.0).clamp(0.0, 1.0);

                transform.rotation = transform.rotation.slerp(target, amount);
            }
        }
    }
}
```

### Native Trigger Haptic

```rust
pub struct SimpleHapticScript;

impl FuckScript for SimpleHapticScript {
    fn on_xr_input(&mut self, ctx: &mut ScriptContext, event: &XrInputEvent) {
        if event.phase != XrInputPhase::Pressed {
            return;
        }

        if event.control != Some(XrInputControl::Trigger) {
            return;
        }

        let Some(hand) = event.hand else {
            return;
        };

        ctx.request_haptic(hand, 0.5, 0.05);
    }
}
```

## 14. Registering A New Native Preset

If a script must run now, make it native.

Steps:

1. Add the script struct in `src/world/scripting.rs`.
2. Implement `FuckScript`.
3. Register it in `GameWorld::new()` in `src/world/mod.rs`.
4. Add the name to `BUILTIN_SCRIPT_NAMES` in `src/world/scripting.rs`.
5. Add tests if the behavior matters.

Example registry line:

```rust
reg.register("MoveUp", || MoveUpScript);
```

## 15. Scene Metadata Rules

`SceneEntity` has these script fields:

```text
script: Option<String>
scripts: Vec<String>
script_components: Vec<ScriptComponent>
```

Use `script_components` as the real editable stack.

`ScriptComponent` has:

```text
name
enabled
source
compile_mode
cache_key
```

Compile modes:

```text
BuiltinNative
EditableNativeCache
CustomNativeCache
```

Rules:

- Built-in script slots usually use `BuiltinNative`.
- Custom editable slots use `CustomNativeCache`.
- Disabled slots stay in the editor but are not sent in `SetScripts`.
- Call `sync_legacy_script_fields()` after changing scene script components.
- Clamp to 32 scripts.

## 16. Export Cache Rules

Export writes script cache data here:

```text
script_cache/script_manifest.json
script_cache/generated_script_bindings.rs
```

Agents must extend this path. Do not invent a second script cache system.

The cache stores:

```text
scene path
entity id
slot index
script name
enabled state
compile mode
source hash
cache key
native symbol
```

## 17. Common Mistakes To Avoid

Do not write this:

```fuckscript
function update() {
}
```

Do not write this:

```javascript
class Enemy extends Script {
}
```

Do not write this:

```lua
function on_update(ctx)
end
```

Do not write this:

```csharp
void Update() {
}
```

Do write this:

```fuckscript
script CustomFuckScript {
    on_update(ctx) {
    }
}
```

## 18. Small Agent Checklist

Before writing:

1. Can a built-in script do it?
2. Does it need to run now? If yes, use native Rust.
3. Is it just saved/exported custom logic? If yes, use editable source.
4. Is the script under 32 slots?
5. Is the hook name allowed?
6. Did you avoid invented helpers?

Before finishing:

1. Keep script order.
2. Keep existing scripts unless asked.
3. Sync legacy fields.
4. For Rust changes, run `cargo check --bins`.
5. Prefer `cargo test --lib` if scripting behavior changed.
